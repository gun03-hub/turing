About Flag The Code - Bug Hunt Challenge

Flag the Code is a technical Bug Hunt competition where participants analyze a deliberately vulnerable website to identify bugs, hidden flaws, and security issues. Teams must explore the website, inspect its code, and report discovered bugs within the given time limit. The team that successfully identifies the maximum valid bugs with proper explanation will be declared the winner.

Event Details

Rules & Guidelines

  • Participation will be in teams of 2 members.
  • Teams will receive access to a demo website containing intentional bugs.
  • Participants must identify bugs such as hidden data, broken authentication, validation errors, or security vulnerabilities.
  • Bugs must be submitted through the Google Form provided by organizers.
  • Each submission must include location, bug type, and explanation.
  • Participants may use browser developer tools (Inspect Element, Console, Network).
  • External hacking tools or attacking real systems is strictly prohibited.

Disqualification Criteria

  • Attempting to attack systems outside the demo website.
  • Sharing answers with other teams.
  • Tampering with the website or event infrastructure.
  • Using automated vulnerability scanners or hacking software.
  • Any unfair or unethical activity during the event.

Note

  • Participants should bring their own laptops.
  • Basic knowledge of HTML, JavaScript, and web security may help.

Format & Winning Criteria

Format

  • The competition will be conducted in a single round bug hunting format.
  • Teams will receive access to a buggy website with multiple pages.
  • The event duration will be approximately 90–120 minutes.

Device & Connectivity

  • Participants must bring their own laptops.
  • Internet access will be provided by organizers.
  • Participants should be able to use browser developer tools.
  • No special software installation is required.

Scoring System

  • Easy Bug – 50 points
  • Medium Bug – 100 points
  • Advanced Bug – 200 points
  • Points are awarded based on bug difficulty.

Winning Criteria

  • The team with the highest total score will be declared the winner.
  • If scores tie, the team discovering the highest difficulty bug wins.
  • If still tied, earliest submission time will be considered.
